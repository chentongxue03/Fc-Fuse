# FC-Fuse

Fc-Fuse is a TensorFlow/Keras project for network traffic classification. It represents each traffic sample with three complementary feature groups: statistical flow features, packet-length sequence features, and payload byte sequence features. These feature groups are encoded by separate neural branches and then fused for final intrusion traffic classification.

The repository contains the code needed for data preprocessing, model construction, training, early-stopping based model saving, test-set evaluation, classification report generation, and confusion matrix visualization.

## Dataset Description

This project uses two public IoT intrusion detection datasets as the experimental data sources: Bot-IoT and CIC-IoT-2023. Both original datasets are distributed in PCAP format and are suitable for building and evaluating multimodal intrusion detection models. In this repository, the model expects preprocessed CSV files derived from these datasets.

The local CSV files used by this project are:

- `file/botiot_last_datasets.csv`: the default training file used by `main/main.py`; it is based on the Bot-IoT dataset.
- `file/CIC_bl_ordered.csv`: an alternative CSV file prepared from the CIC-IoT-2023 dataset for replacement or comparative experiments.

The CSV files are not included in this GitHub repository because they are large dataset artifacts. They are excluded by the `.gitignore` rule:

```text
file/*.csv
```

Before running the project, place the required CSV files in the local `file/` directory with the expected file names.

### Bot-IoT

The Bot-IoT dataset was constructed by the University of New South Wales (UNSW). It simulates a typical IoT network environment containing devices such as temperature and humidity sensors, smart appliances, and other IoT nodes. The dataset covers four major attack categories:

- DDoS
- DoS
- Scan
- Theft

In this project, the default CSV file `file/botiot_last_datasets.csv` is used as the Bot-IoT-based training dataset.

### CIC-IoT-2023

The CIC-IoT-2023 dataset was released by the Canadian Institute for Cybersecurity (CIC) in 2023. It captures both benign and malicious traffic generated by 105 real IoT devices in household and industrial environments. The dataset includes seven major attack categories:

- DDoS
- DoS
- Recon
- Web-based
- Brute Force
- Spoofing
- Mirai Botnet

In this project, the local file `file/CIC_bl_ordered.csv` is kept as the CIC-IoT-2023-based CSV dataset for alternative experiments.

## Expected CSV Schema

The preprocessing code in `main/data_pressor.py` assumes that each CSV file contains at least the following columns:

| Column | Purpose | Processing |
| --- | --- | --- |
| `category` | Class label | Encoded with `LabelEncoder` |
| `pktlen` | Packet-length sequence | Split by commas, converted to integers, and used as the PKT branch input after absolute-value transformation |
| `fwd_payload` | Forward payload | Parsed from a hexadecimal string into a byte sequence |
| `bwd_payload` | Backward payload | Parsed from a hexadecimal string into a byte sequence |

The following columns are excluded from the statistical feature group:

```text
Uuid, pktlen, fwd_payload, bwd_payload, category, subclass
```

All remaining CSV columns are treated as statistical flow features. Missing values are filled with `0`, and the feature matrix is standardized with `StandardScaler` before being passed into the STATS branch.

Example `pktlen` value:

```text
60,60,-52,1500,40
```

Example payload value:

```text
4500003c1c4640004006
```

If a payload field is empty or uses `-`, the preprocessing code converts it to a zero-filled sequence.

## Feature Processing

Each traffic sample is converted into three model inputs:

1. **STATS features**: numerical flow-level statistics from the CSV columns that are not excluded.
2. **PKT features**: packet-length sequences parsed from `pktlen`; the default maximum sequence length is `20`.
3. **PYL features**: byte sequences parsed from `fwd_payload` and `bwd_payload`; the default maximum payload length is `50` bytes per direction, and the two directions are concatenated.

## Model Architecture

The model contains the following components:

- `STATS_NN`: dense layers for statistical feature encoding.
- `PKT_NN`: embedding and bidirectional LSTM layers for packet-length sequence encoding.
- `PYL_NN`: embedding, 1D convolution, and global max pooling layers for payload byte sequence encoding.
- `FusionMLP`: dense fusion module for combining the three feature embeddings.
- `ClassifierModel`: final softmax classifier.

## Usage

The default training entry point reads:

```text
../file/botiot_last_datasets.csv
```

Run training from the `main` directory:

```bash
cd main
python main.py
```

To use `CIC_bl_ordered.csv` or another compatible CSV file, update `csv_path` in `main/main.py`:

```python
csv_path = "../file/CIC_bl_ordered.csv"
```

Key training parameters are defined in `main/main.py`:

```python
epochs = 50
batch_size = 256
alpha = 1.0
beta = 0.5
margin = 1.0
learning_rate = 1e-3
patience = 5
```

## Outputs

During training, the script prints per-epoch loss values and accuracy metrics. When validation accuracy improves, the model components are saved under:

```text
main/logs/<timestamp>/best_model_epoch*_acc*
```

After training, the best model is loaded and evaluated on the test set. The generated outputs include:

- test accuracy
- `classification_report.txt`
- `confusion_matrix.png`
- `logs/<timestamp>/training_losses.png`

`classification_report.txt` and `confusion_matrix.png` are generated in the current working directory. If training is launched from `main/`, they will appear in the `main/` directory.
